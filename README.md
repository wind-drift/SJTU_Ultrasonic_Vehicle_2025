# SJTU超声波跟随小车2025

## 使用方法：

首先，你需要安装好`PlatformIO`插件，`PlatformIO`插件的**安装及使用方法详见文末**。如果你希望使用 $\LaTeX$ 撰写实验报告，你还需要安装`TexLive`编译环境和`LaTeX Workshop`插件。MCU、电机控制板、超声波模块、电池模块等硬件材料老师会给准备好的。

将这个项目克隆到本地，在文件夹中双击 `open.bat` 文件，会自动打开 `vscode` 并加载项目，注意路径中最好不要含有中文字符。

项目中有三个文件夹，`Lab_Report`用来存放实验报告，`Ultrasonic_Vehicle_2025`是小车部分的控制代码，`Ultrasonic_Transmitter`是超声波发生器的代码。

打开 `vscode` 后（假设你已经启用`PlatformIO`插件），你应该会看到左下角有一个小房子图标，点击进入`PlatformIO`的主界面，用`PlatformIO`打开项目（如果没有这个选项，可能你需要查一下怎么打开已有项目），然后等待初始化完成。

接下来你就可以写代码了，如果有什么不会的，或者代码看不懂的地方就问问AI吧~

当然，也欢迎你在我的邮箱`tingjiang@sjtu.edu.cn`留言。

## 项目介绍

### 电路设计：

本部分由 Moir_Z 完成，将调制正弦波（也就是超声波接收器收到超声波后产生的信号）转换为 $40kHz$ 的脉冲信号。

### 程序设计：

- `main.cpp` 主程序入口
- `main_controller.h/.cpp` 作为上层控制器，负责集成和处理底层信号并控制小车运行
- `motor_controller.h/.cpp` 作为电机控制板底层，负责驱动电机基础运动
- `pid_controller.h/.cpp` PID算法库，将超声波接收的时间差转化为电机控制命令。具体原理可以看代码注释或者去bilibili大学学习
- `ultrasonic_controller.h/.cpp` 作为超声波模块底层，负责驱动超声波接收器模块，计算两超声波接收器接受到信号的时间差，并对原始数据进行软件滤波和映射处理（将`timeDiff`映射到\[-1, 1\]），方便后期 PID 控制
- `config.h` 用来存储所有的宏定义，比如引脚编号、PID系数等常量。直接调用宏定义而非数字，这样可以方便后期调试时修改

### 关于代码的上传

**每次修改代码之前，先在源代码管理中点击“拉取”，同步最新进度**。

提交代码的时候，在`vscode`中右键点击`Ultrasonic_Vehicle_2025`文件夹，选择“在集成终端中打开”选项。

在终端输入

```bash
git add .
git commit -m "anything you want to say!"
```

然后快捷键`Ctrl+Shift+G`打开源代码管理，点击提交。如果无法正常提交代码，可以尝试添加SSH秘钥，具体方式请询问AI。或者你也可以尝试关闭SJTU网，然后连接手机流量热点提交，有可能是校园网限制的问题。

如果你的小伙伴不擅长科学上网，你也可以考虑使用国内的Gitee进行小组合作。
